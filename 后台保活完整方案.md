# 后台保活完整方案

## 问题描述

用户在使用应用时，如果切换到其他应用（如微信、浏览器），再切回来时：
- ❌ **AI 正在输出的回答被中断**
- ❌ **应用更新下载被中断**
- ❌ **其他异步任务被暂停或终止**

## 解决方案

### 1. 核心技术

#### 1.1 WakeLock（保持设备唤醒）
- 防止设备休眠
- 防止系统杀掉应用进程
- 在任务执行期间自动启用，完成后自动释放

#### 1.2 后台下载服务（flutter_downloader）
- 使用 Android 系统级下载管理器
- 即使应用切到后台，下载也会继续
- 显示系统通知，用户可以看到下载进度

#### 1.3 Android Manifest 配置
- `WAKE_LOCK` 权限：允许保持设备唤醒
- `FOREGROUND_SERVICE` 权限：允许运行前台服务
- `ACCESS_NETWORK_STATE` 权限：监控网络状态
- Activity 保活配置：防止系统清除应用状态

### 2. 已实现的功能

#### 2.1 后台任务服务（BackgroundTaskService）

创建了统一的后台任务管理服务，支持：
- **AI 流式输出保活**：聊天时 AI 正在输出，切换应用不会中断
- **文件下载保活**：应用更新下载时，切换应用继续下载
- **其他异步任务保活**：可扩展支持更多场景

**使用方式：**
```dart
// 开始任务
await BackgroundTaskService.instance.startTask(BackgroundTaskType.aiStreaming);

// 执行任务...

// 结束任务
await BackgroundTaskService.instance.endTask(BackgroundTaskType.aiStreaming);
```

#### 2.2 AI 聊天保活

修改了 `lib/providers/chat_provider.dart`：
- 流式输出开始时自动启用 WakeLock
- 流式输出结束时自动释放 WakeLock
- 用户切换应用时，AI 继续输出
- 切回应用时，可以看到完整的回答

#### 2.3 应用更新保活

修改了 `lib/services/update_service.dart`：
- 使用系统级后台下载服务
- 下载时显示系统通知
- 切换应用时继续下载
- 下载完成后自动打开安装界面

### 5. 技术细节

#### 5.1 WakeLock 工作原理

```dart
// 启用 WakeLock
await WakelockPlus.enable();
// 此时系统不会：
// - 让设备休眠
// - 杀掉应用进程（除非内存严重不足）
// - 暂停网络连接

// 释放 WakeLock
await WakelockPlus.disable();
// 恢复正常的电源管理
```

#### 5.2 任务计数机制

```dart
// 支持多个任务同时运行
await BackgroundTaskService.instance.startTask(BackgroundTaskType.aiStreaming);
await BackgroundTaskService.instance.startTask(BackgroundTaskType.downloading);

// 只有所有任务都结束时，才释放 WakeLock
await BackgroundTaskService.instance.endTask(BackgroundTaskType.aiStreaming);
// WakeLock 仍然启用，因为还有 downloading 任务

await BackgroundTaskService.instance.endTask(BackgroundTaskType.downloading);
// 现在所有任务都结束了，自动释放 WakeLock
```

#### 5.3 后台下载通知

使用 `flutter_downloader` 时，系统会自动显示通知：
- 显示下载进度（百分比）
- 显示下载速度
- 用户可以点击通知查看详情
- 下载完成后可以点击通知安装

### 6. 注意事项

#### 6.1 电池消耗

WakeLock 会增加电池消耗，因此：
- ✅ 只在必要时启用（任务执行期间）
- ✅ 任务完成后立即释放
- ✅ 使用任务计数，避免过早释放
- ❌ 不要长时间持有 WakeLock

#### 6.2 系统限制

即使有 WakeLock，以下情况仍可能被杀：
- 内存严重不足（系统强制回收）
- 用户手动清理后台
- 省电模式下的激进策略
- 特定厂商的后台管理（小米、华为等）

#### 6.3 用户设置

建议在应用内添加提示，引导用户：
1. 将应用加入白名单
2. 关闭电池优化
3. 允许后台运行

不同厂商设置方法见：`防止后台被杀说明.md`

### 7. 扩展支持

如果需要支持更多后台任务，只需：

1. 在 `BackgroundTaskType` 枚举中添加新类型：
```dart

  uploading,    // 新增

```

2. 在任务开始时调用：
```dart

```

3. 在任务结束时调用：
```dart
await BackgroundTaskService.instance.endTask(BackgroundTaskType.uploading);
```

### 9. 常见问题

**Q: 为什么还是被杀了？**
A: 检查：
- 是否在任务开始时调用了 `startTask`
- 是否在任务结束时调用了 `endTask`
- 手机是否开启了省电模式
- 是否需要在手机设置中添加白名单

**Q: 电池消耗会增加多少？**
A: 取决于任务时长：
- AI 输出（10-30秒）：几乎无影响
- 应用更新下载（1-3分钟）：轻微影响
- 长时间任务（>10分钟）：明显影响

**Q: 可以永久保持后台吗？**
A: 不建议，也不可能：
- Android 系统会限制长时间后台运行
- 会严重影响电池续航
- 用户体验不好

**Q: 如何测试是否生效？**
A: 最简单的方法：
1. 开始 AI 对话
2. 立即切换应用
3. 等待 20 秒
4. 切回来看是否有完整回答

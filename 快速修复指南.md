# 快速修复指南

## 问题 1：更新日志乱码

### 一键修复（推荐）

在 PowerShell 中运行：

```powershell
.\quick_fix_encoding.ps1
```

输入服务器密码后，脚本会自动：
1. 上传修复脚本到服务器
2. 执行修复
3. 验证结果

### 手动修复

如果自动脚本失败，可以手动操作：

```bash
# 1. 上传脚本
scp fix_server_encoding.sh admin@47.104.165.105:/tmp/

# 2. SSH 登录
ssh admin@47.104.165.105

# 3. 运行脚本
bash /tmp/fix_server_encoding.sh
```

### 验证

修复后，访问以下地址应该看到正确的中文：
http://47.104.165.105:8000/api/app/version

## 问题 2：切换应用后被杀后台

### 重新构建应用

已经修改了 Android 配置，需要重新构建 APK：

```powershell
# 清理旧构建
flutter clean

# 获取依赖
flutter pub get

# 构建新 APK
flutter build apk --release
```

构建完成后，APK 位于：
```
build/app/outputs/flutter-apk/app-release.apk
```

### 安装测试

1. 将新 APK 安装到手机
2. 打开应用
3. 切换到其他应用（如微信、浏览器）
4. 等待几分钟
5. 返回应用，检查是否保持状态

### 用户设置（重要）

由于 Android 系统限制，还需要用户在手机上设置：

**小米手机**：
```
设置 → 应用设置 → 应用管理 → 伴学 
→ 省电策略 → 无限制
→ 自启动 → 开启
```

**华为手机**：
```
设置 → 应用 → 应用启动管理 → 伴学 → 手动管理
→ 允许自启动
→ 允许关联启动
→ 允许后台活动
```

**OPPO 手机**：
```
设置 → 电池 → 应用耗电管理 → 伴学 → 允许后台运行
设置 → 应用管理 → 伴学 → 权限 → 自启动 → 开启
```

**VIVO 手机**：
```
设置 → 电池 → 后台高耗电 → 伴学 → 允许
设置 → 更多设置 → 权限管理 → 自启动 → 伴学 → 开启
```

## 常见问题

### Q: 修复脚本执行失败？
A: 检查：
- 服务器 SSH 连接是否正常
- 密码是否正确
- 服务器路径是否正确：`/home/admin/study-assistant-app`

### Q: 应用还是被杀后台？
A: 这是正常的，Android 系统限制很多：
- 确保用户已按照上述步骤设置手机
- 不同品牌手机效果不同
- 内存不足时系统仍会杀掉应用

### Q: 如何查看服务器日志？
A: SSH 登录后：
```bash
tail -f /tmp/uvicorn.log
```

### Q: 如何重启服务器服务？
A: SSH 登录后：
```bash
pkill -f 'uvicorn main:app'
cd /home/admin/study-assistant-app/backend
nohup venv/bin/python3 venv/bin/uvicorn main:app --host 0.0.0.0 --port 8000 > /tmp/uvicorn.log 2>&1 &
```

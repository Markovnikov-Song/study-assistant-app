# 防止应用后台被杀配置说明

## 已完成的配置

### 1. AndroidManifest.xml 配置优化

已在 `android/app/src/main/AndroidManifest.xml` 中添加以下配置：

#### 新增权限
```xml
<!-- 防止后台被杀 -->
<uses-permission android:name="android.permission.WAKE_LOCK"/>
<uses-permission android:name="android.permission.FOREGROUND_SERVICE"/>
```

#### Application 级别配置
```xml
android:allowBackup="true"
android:supportsRtl="true"
android:persistent="false"
```

#### Activity 级别配置
```xml
android:excludeFromRecents="false"      <!-- 不从最近任务中排除 -->
android:alwaysRetainTaskState="true"    <!-- 始终保持任务状态 -->
android:clearTaskOnLaunch="false"       <!-- 启动时不清除任务 -->
android:stateNotNeeded="false"          <!-- 需要保存状态 -->
```

## 配置说明

### 这些配置的作用

1. **WAKE_LOCK 权限**：允许应用在屏幕关闭时保持 CPU 运行（需要时使用）
2. **FOREGROUND_SERVICE 权限**：允许应用运行前台服务（如果需要长期后台任务）
3. **excludeFromRecents="false"**：确保应用显示在最近任务列表中
4. **alwaysRetainTaskState="true"**：系统不会清除应用的任务状态
5. **clearTaskOnLaunch="false"**：重新启动时保持之前的状态
6. **stateNotNeeded="false"**：告诉系统需要保存和恢复状态

## 注意事项

### 1. 系统限制
即使有这些配置，Android 系统仍可能在以下情况下杀死应用：
- 内存严重不足
- 用户手动清理后台
- 省电模式下
- 特定厂商的激进内存管理（如小米、华为、OPPO）

### 2. 厂商特殊处理
不同手机厂商有不同的后台管理策略：

**小米 MIUI**：
- 设置 → 应用设置 → 应用管理 → 伴学 → 省电策略 → 无限制
- 设置 → 应用设置 → 应用管理 → 伴学 → 自启动 → 开启

**华为 EMUI**：
- 设置 → 应用 → 应用启动管理 → 伴学 → 手动管理
- 允许自启动、允许关联启动、允许后台活动

**OPPO ColorOS**：
- 设置 → 电池 → 应用耗电管理 → 伴学 → 允许后台运行
- 设置 → 应用管理 → 伴学 → 权限 → 自启动 → 开启

**VIVO**：
- 设置 → 电池 → 后台高耗电 → 伴学 → 允许
- 设置 → 更多设置 → 权限管理 → 自启动 → 伴学 → 开启

### 3. 用户指导
建议在应用内添加提示，引导用户：
1. 将应用加入白名单
2. 关闭电池优化
3. 允许后台运行

## 进一步优化（可选）

如果需要更强的后台保活能力，可以考虑：

### 1. 前台服务（适用于需要持续运行的场景）
需要在 Dart 代码中实现前台服务，显示持续通知。

### 2. WorkManager（适用于定时任务）
使用 Android WorkManager 进行可靠的后台任务调度。

### 3. 保活插件
可以考虑使用 Flutter 插件如：
- `flutter_foreground_task`：前台服务
- `workmanager`：后台任务调度
- `android_alarm_manager_plus`：定时任务

## 测试方法

1. **基本测试**：
   - 打开应用
   - 切换到其他应用
   - 等待 5-10 分钟
   - 返回应用，检查状态是否保持

2. **压力测试**：
   - 打开应用
   - 打开多个其他应用（消耗内存）
   - 返回应用，检查是否被杀

3. **长时间测试**：
   - 打开应用
   - 锁屏或切换到其他应用
   - 等待 1-2 小时
   - 返回应用，检查状态

## 第一章　需求分析

### 1.1 背景与痛点

当前学生学习工具存在三大痛点：①问答工具（如 ChatGPT）无法结合个人资料，回答泛化；②复习工具（如 Anki）与学习内容割裂，需手动制卡；③学习规划工具（如 Notion）无法感知知识掌握状态，计划与实际脱节。

### 1.2 竞品分析

| 维度 | 学伴 | 有道精品课 | Anki | Notion AI |
|------|------|-----------|------|-----------|
| 基于个人资料问答 | ✅ RAG 三模式 | ❌ | ❌ | ⚠️ 有限 |
| 自动生成思维导图 | ✅ | ❌ | ❌ | ❌ |
| 边界控制出题 | ✅ 前/当/后置知识区 | ⚠️ 固定题库 | ❌ | ❌ |
| SM-2 间隔复习 | ✅ 改良版 | ❌ | ✅ 标准版 | ❌ |
| Multi-Agent 学习规划 | ✅ | ❌ | ❌ | ❌ |
| 自然语言指令 | ✅ CAS 框架 | ❌ | ❌ | ⚠️ 有限 |

### 1.3 目标用户

中学生、大学生，有系统学习某学科的需求，希望 AI 辅助而非替代学习。

### 1.4 主要功能

- **AI 问答**：基于上传资料的 RAG 检索增强问答，支持严谨/宽泛/解题三种模式
- **思维导图**：自动生成知识树，支持手动编辑、节点讲义生成
- **练习出题**：边界控制出题，错题自动入库
- **间隔复习**：改良 SM-2 算法，学科差异化调度
- **学习规划**：Multi-Agent 生成跨学科学习计划，与日历双向同步
- **学习日历**：事件管理、番茄钟、学习统计

```

```
学伴
├── 聊天模块（chat）

## 第三章　详细设计

### 3.1 界面设计（典型流程）

**学习流程**：
1. 上传资料（PDF/DOCX/PPTX）→ 自动分块向量化
2. 生成思维导图 → 点击节点 → 生成讲义
3. 讲义底部「去练习」→ 边界控制出题 → 答错自动入错题本
4. 错题本 → 复盘 → SM-2 调度下次复习
5. 聊天室自然语言「帮我制定学习计划」→ CAS 触发 Multi-Agent 规划

### 3.2 数据库设计（核心表）

### 3.3 关键算法

#### 3.3.1 改良 SM-2 间隔重复算法

标准 SM-2 不区分学科特性，本系统引入三个修正系数：

```
interval_new = interval_old × ease_factor/100 × subject_decay × difficulty_factor × quality_factor

subject_decay（学科衰减）：
  数学/物理 = 0.85（公式容易遗忘）
  英语      = 1.15（语感衰退慢）
  历史/地理 = 0.95（理解性记忆）

difficulty_factor：简单 1.3x / 中等 1.0x / 困难 0.7x

quality_factor（用户评分 0-3）：
  0（完全忘）= 0 → 立即复习
  1（模糊）  = 0.5x
  2（想起）  = 1.0x
  3（巩固）  = 1.3x
```

掌握度量化（0-5 分）：
```
mastery = min(5, repetitions × 0.8 + (ease_factor/100 - 1.3) × 2)
```

#### 3.3.2 RAG 检索增强生成

```
用户问题 → text-embedding-3-small 向量化
         → PGVector cosine 相似度搜索（阈值 0.7，Top-K=8）
         → 三模式处理：
           strict：仅基于检索结果回答
           broad：检索结果 + 通用知识，标注来源
           solve：结构化解题（考点→思路→步骤→踩分点→易错点）
         → SSE 流式输出 + sources 返回
```

#### 3.3.3 边界控制出题

```
出题范围 = 前置知识区（20%）+ 当前知识区（60%）+ 后置知识区（20%）
难度分布 = L1基础（40%）+ L2中等（40%）+ L3进阶（20%）
判题：选择/判断 → 精确匹配；填空/计算 → 模糊匹配（去空格、标点、大小写）
答错 → 自动创建错题记录 + SM-2 复习卡片
```

#### 3.3.4 三层学习进度

```
overall = read × 0.3 + practice × 0.5 + mastery × 0.2

read     = lit_nodes / total_nodes
practice = (reviewed_cards + total_cards × 0.3) / (total_cards × 1.3)
mastery  = avg_mastery_score / 5.0
```

#### 3.3.5 CAS 受控动作空间框架

```
用户输入 → IntentMapper（LLM，超时 3s 降级为 RuleMapper）
         → ActionRegistry 验证 action_id
         → 参数完整性校验（缺参 → param_fill 卡片）
         → ExecutorRegistry.run()（捕获所有异常，返回 fallback）
         → ActionResult（text/card/navigate/modal）
```

### 4.2 技术指标

| 指标 | 数值 |
|------|------|
| 聊天响应首 token 延迟 | < 2s（SSE 流式） |
| 思维导图生成时间 | 10-30s（依赖 LLM） |
| 讲义生成时间 | 15-60s（流式，首字符 < 3s） |

## 第五章　安装及使用

### 5.1 环境要求

**后端**：Python 3.11+、PostgreSQL 15+（含 pgvector 扩展）

**前端**：Flutter 3.x、Android SDK 或 Xcode

### 5.2 后端安装

```bash
cd backend
pip install -r requirements.txt
cp .env.example .env
# 编辑 .env，填写 DATABASE_URL、LLM_API_KEY 等
python -m uvicorn main:app --host 0.0.0.0 --port 8000 --reload
```

### 5.3 前端安装

```bash
flutter pub get
# USB 调试（推荐）
flutter run --dart-define=API_BASE_URL=http://localhost:8000
# WiFi 调试
flutter run --dart-define=API_BASE_URL=http://192.168.x.x:8000
```

### 5.4 一键启动（开发调试）

```powershell
# 项目根目录
.\dev.ps1
# 自动检测 USB/热点场景，同时启动后端和前端
```

### 5.5 典型使用流程

1. 注册/登录 → 创建学科 → 上传资料
2. 课程空间 → 生成思维导图 → 点击节点 → 生成讲义
3. 讲义页底部「去练习」→ 完成练习
4. 工具箱 → 复盘中心 → 复盘错题
5. 聊天室 → 输入「帮我制定学习计划」→ 确认规划

## 第六章　项目总结

### 6.1 技术收获

- 掌握了 RAG 检索增强生成的工程实现，包括文档分块、向量化、相似度检索全链路
- 深入理解 SM-2 间隔重复算法，并针对学科特性进行了改良
- 设计并实现了 CAS 受控动作空间框架，解决了 LLM 指令越界问题
- 实践了 Multi-Agent 协作规划系统的设计与实现

### 6.2 克服的困难

- **流式响应与状态管理**：SSE 流式输出与 Riverpod 状态管理的协调，通过 StreamController + AsyncValue 解决
- **类型安全**：后端 PostgreSQL 返回字符串 ID 导致前端崩溃，引入统一的安全解析工具函数
- **多模块同步**：日历、规划、思维导图三个模块的状态同步，通过 EventBus 广播机制解决

### 6.3 后续升级方向

- 接入真实支付 SDK，实现 Token 套餐购买
- 实现 FCM/APNs 远程推送通知
- 增量重规划（用户调整截止日期后自动重新排期）
- 技能市场（用户分享自定义学习方法）

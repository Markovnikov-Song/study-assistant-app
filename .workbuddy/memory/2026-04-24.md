## 错题本页面崩溃修复：`type 'String' is not a subtype of type 'int'`

**问题**：进入错题本页面立刻报错 `type 'String' is not a subtype of type 'int'`

**根因**：`notebook_service.dart` 第67行 `section['subject_id'] as int?` 做了不安全的类型转换。当 JSON 中 `subject_id` 的值不是严格的 int 类型（可能被序列化为字符串或 num），直接 `as int?` 会崩溃。

**修复**：
1. `lib/services/notebook_service.dart` — 将 `section['subject_id'] as int?` 改为 `(rawKey as num).toInt()` 安全转换
2. `lib/services/history_service.dart` — 同样修复 `json['subject_id'] as int?` → `(json['subject_id'] as num?)?.toInt()`
3. `backend/routers/notebooks.py` — 同步 NoteOut 模型为完整版本，添加 mastery_score/review_count/last_reviewed_at/mistake_category/mastery_history 字段（之前 notebooks.py 的 NoteOut 是旧版，缺少这些字段，与 notes.py 不一致）
4. `backend/migrations/009_add_mistake_enhanced_fields.sql` — 创建缺失的数据库迁移，添加 mastery_score 等5个字段

**经验**：Dart 中 JSON 反序列化时，所有 int 类型字段都应用 `(json['xxx'] as num?)?.toInt()` 而非 `as int?`，避免类型不匹配崩溃。

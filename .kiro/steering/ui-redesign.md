# UI 重设计规范文档

## 一、设计目标

将现有"学科列表 → 学科详情"两级结构重构为以**功能为核心**的一级导航，
学科作为全局上下文切换器存在，而非入口页面。

---

## 二、整体导航结构

### 底部导航栏（5个 Tab）

| 序号 | Tab 名称 | 图标 | 说明 |
|------|----------|------|------|
| 1 | 问答 | chat_bubble_outline | RAG 问答，strict/broad 模式 |
| 2 | 解题 | calculate_outlined | 结构化解题输出 |
| 3 | 导图 | account_tree_outlined | 思维导图生成 |
| 4 | 出题 | auto_awesome_outlined | 预测试卷 + 自定义出题 |
| 5 | 我的 | person_outline | 个人中心 |

> 原"资料管理"和"历年题"不作为独立 Tab，整合进"我的"页面或学科管理抽屉。

---

## 三、学科切换器（顶部 SubjectBar）

所有功能页（问答/解题/导图/出题）顶部统一显示一个**学科选择栏**：

```
┌─────────────────────────────────────────┐
│  📚 材料力学  ▾          + 新建学科      │
└─────────────────────────────────────────┘
```

- 点击学科名称 → 弹出底部抽屉，列出所有学科，可切换/新建/管理
- 学科状态全局共享（用 Riverpod `currentSubjectProvider`）
- 未选学科时显示"请先选择学科"引导

### 学科抽屉内容
- 学科列表（置顶的排前面）
- 每项右侧有"⋯"菜单（编辑/置顶/归档/删除）
- 底部"+ 新建学科"按钮
- 归档学科折叠展示

```
├─────────────────────────────────────────┤
│  [历史记录 ▾]              [+ 新建对话]  │  ← SessionBar
├─────────────────────────────────────────┤
│                                         │
│           消息气泡列表                   │
│                                         │

│  □ 结合通用知识                          │  ← 选项行

└─────────────────────────────────────────┘
```

### 4.2 解题页（SolvePage）

结构同问答页，区别：
- 输入框 placeholder 为"输入题目…"
- AI 回复按「考点 → 解题思路 → 解题步骤 → 踩分点 → 易错点」结构化展示
- 无"结合通用知识"选项（解题固定用 solve 模式）

### 4.3 思维导图页（MindMapPage）

```
┌─────────────────────────────────────────┐
│  📚 材料力学  ▾                          │
├─────────────────────────────────────────┤
│  选择资料范围：[全部资料 ▾]              │
├─────────────────────────────────────────┤
│                                         │
│         思维导图渲染区域                 │
│      （Markdown 文本 / WebView）         │
│                                         │
├─────────────────────────────────────────┤
│  [历史导图 ▾]    [生成导图]  [导出 MD]  │
└─────────────────────────────────────────┘
```

### 4.4 出题页（QuizPage）

两个子 Tab：
- **预测试卷**：一键生成，展示 Markdown，可导出
- **自定义出题**：题型多选 + 各题型数量/分值 + 难度 + 考点

## 五、学科管理页（SubjectsPage，从"我的"进入）

- 保留现有学科列表 UI
- 每个学科卡片点击 → 进入**学科资料页**
- 学科资料页包含两个 Tab：
  - **资料库**（原"资料管理"）：上传/查看/删除学习资料
  - **历年题**：上传/查看/删除历年题文件及题目

> 资料库和历年题是包含关系（历年题是一种特殊资料），
> 但功能差异较大（历年题需要 AI 结构化解析），故分 Tab 展示而非合并。

## 六、路由结构

```
/login                    登录
/register                 注册
/                         主页（底部导航 Shell）
  /chat                   问答
  /solve                  解题
  /mindmap                思维导图
  /quiz                   出题
  /profile                我的
    /subjects             学科管理
      /subjects/:id       学科资料页（资料库 + 历年题）
    /history              对话历史
```

---

## 七、全局状态

| Provider | 类型 | 说明 |
|----------|------|------|
| `currentSubjectProvider` | `StateProvider<Subject?>` | 当前选中学科，全局共享 |
| `authProvider` | `StateNotifierProvider<AuthNotifier, AuthState>` | 登录状态 |
| `subjectsProvider` | `FutureProvider<List<Subject>>` | 学科列表 |
| `chatProvider(subjectId)` | `StateNotifierProviderFamily` | 各学科问答状态 |

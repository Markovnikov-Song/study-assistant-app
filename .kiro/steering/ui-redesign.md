# UI 重设计规范文档（对话中心版）

## 一、设计目标

将应用重构为以**对话为主入口**的交互架构，类比豆包式体验。
底部 4 个 Tab 作为主导航，对话页为默认首页，其余功能通过 Tab 直达或对话触发。

---

## 二、整体导航结构

### 底部导航栏（4 个 Tab）

| 序号 | Tab 名称 | 图标 | 路由 | 说明 |
|------|----------|------|------|------|
| 1 | 答疑室 | chat_bubble_outline | `/` | 通用聊天页，默认首页，场景识别入口 |
| 2 | 图书馆 | menu_book_outlined | `/course-space` | 学科大纲 + 思维导图 + 讲义 |
| 3 | 工具箱 | edit_outlined | `/toolkit` | 错题本 + 笔记本 + 解题 + 出题 + 思维导图 |
| 4 | 我的 | person_outline | `/profile` | 个人中心、学科管理、对话历史 |

> 思维导图有两个入口：① 图书馆 → 学科卡片内直接生成；② 工具箱→ 选择学科后创建。

## 四、场景识别触发的 UI 流程（对话页内）

### 4.1 学科意图识别

用户输入学科相关问题时，对话流中弹出确认卡片：

```
┌─────────────────────────────────────────┐
│  🎓 检测到高数相关问题                   │
│  切换到「高数」专属对话？                │
│  [切换]  [继续通用对话]                  │
└─────────────────────────────────────────┘
```

确认后路由跳转至 `/chat/:chatId/subject/:subjectId`。

### 4.2 规划意图识别

用户输入备考/学习目标时，触发多轮追问，完成后推送：

```
┌─────────────────────────────────────────┐
│  📋 已为你生成期末复习计划               │
│  [查看计划]  [稍后再说]                  │
└─────────────────────────────────────────┘
```

确认后路由跳转至 `/chat/:chatId/task/:taskId`。

### 4.3 工具意图识别

用户输入笔记/错题相关需求时，对话流中弹出跳转卡片：

```
┌─────────────────────────────────────────┐
│  📓 跳转到笔记本？                       │
│  [一键跳转]  [在对话中继续]              │
└─────────────────────────────────────────┘
```

跳转后页面右下角显示悬浮返回按钮，操作完成可一键回到原对话。

### 4.4 Spec 模式识别

用户输入大型学习目标时：

```
┌─────────────────────────────────────────┐
│  🗂 检测到大型学习任务                   │
│  启动「Spec 规划模式」？                 │
│  [启动]  [普通对话]                      │
└─────────────────────────────────────────┘
```

确认后路由跳转至 `/spec`。

## 五、各功能页详细设计

### 5.1 通用聊天页 / 学科专属对话页 / 任务对话页

三者复用同一个 ChatPage Widget，通过路由参数区分场景：

```
┌─────────────────────────────────────────┐
│  ← [高数专属对话]              ⋯        │  ← 顶栏（学科/任务对话时显示标题）
├─────────────────────────────────────────┤
│                                         │
│           消息气泡列表                   │
│                                         │
├─────────────────────────────────────────┤
│  📷  🖼  [输入消息…          ]  [发送]  │
└─────────────────────────────────────────┘
```

### 5.2 工具箱页（/toolkit）

桌面应用图标风格，4 列网格排列，每个工具是一个带图标+名称的方形卡片：

```
┌─────────────────────────────────────────┐
│  工具箱                                 │
├─────────────────────────────────────────┤
│                                         │
│  ┌──────┐ ┌──────┐ ┌──────┐ ┌──────┐  │
│  │  📕  │ │  📓  │ │  📐  │ │  �  │  │
│  │错题本│ │笔记本│ │ 解题 │ │ 出题 │  │
│  └──────┘ └──────┘ └──────┘ └──────┘  │
│                                         │
│  （后续可扩展更多工具卡片）              │
│                                         │
└─────────────────────────────────────────┘
```

- 每个卡片点击跳转对应功能页
- 卡片样式：圆角方形图标 + 下方文字标签，与手机桌面 App 图标一致
- 支持后续扩展新工具卡片，无需改动布局

### 5.3 解题页（/solve）

```
┌─────────────────────────────────────────┐
│  ← 解题                                 │
├─────────────────────────────────────────┤
│                                         │
│  AI 回复按结构化格式展示：               │
│  考点 → 解题思路 → 解题步骤             │
│  → 踩分点 → 易错点                      │
│                                         │
├─────────────────────────────────────────┤
│  📷  🖼  [输入题目…          ]  [发送]  │
└─────────────────────────────────────────┘
```

### 5.4 出题页（/quiz）

两个子 Tab：
- **预测试卷**：一键生成，展示 Markdown，可导出
- **自定义出题**：题型多选 + 各题型数量/分值 + 难度 + 考点

### 5.5 课程空间页（/course-space）

```
┌─────────────────────────────────────────┐
│  ← 课程空间                             │
├─────────────────────────────────────────┤
│  学科列表（卡片形式）                    │
│  每个学科卡片点击进入学科详情            │
└─────────────────────────────────────────┘
```

学科详情页包含三个 Tab：
- **大纲**：知识点树形结构
- **思维导图**：基于大纲节点生成，含历史导图列表
- **讲义**：讲义查看与生成

## 七、全局状态

| Provider | 类型 | 说明 |
|----------|------|------|
| `currentSubjectProvider` | `StateProvider<Subject?>` | 当前选中学科，全局共享 |
| `authProvider` | `StateNotifierProvider<AuthNotifier, AuthState>` | 登录状态 |
| `subjectsProvider` | `FutureProvider<List<Subject>>` | 学科列表 |
| `chatProvider(chatId)` | `StateNotifierProviderFamily` | 各对话 Session 状态 |
| `currentSessionProvider` | `StateProvider<Session?>` | 当前活跃 Session |

# Node Lecture - 节点讲义懒加载生成器

## Overview

This skill generates teaching lecture notes for a specific node in a knowledge tree or mind map, using a **lazy-loading** approach. Instead of generating all lectures at once, it produces lecture content only for the node the user explicitly requests. The lecture depth, style, and content are automatically calibrated based on the node's position in the hierarchy.

## Trigger Phrases (触发词)

| 触发词类型 | 示例短语 |
|-----------|---------|
| 直接命令 | "展开讲义"、"写个讲义"、"详细讲一下" |
| 节点指定 | "讲一下欧拉公式"、"展开这个知识点" |
| 层级指定 | "给第三章写个讲义"、"这一节展开讲讲" |
| 英文表达 | "generate lecture for this node", "explain this concept in detail" |

**关键触发条件：**
- 用户指向了思维导图/知识树中的某个具体节点
- 用户请求对某个知识点/章节进行深入讲解
- 用户想要把导图节点转化为可阅读的讲义文档

## Core Concept: Lazy-Loading Lecture Generation (懒加载讲义)

### 设计哲学

```
传统方式（全量生成）：
  书 → 一次性生成全部讲义 → 30000字 → 用户根本看不完 ❌

懒加载方式（按需生成）：
  用户点击"欧拉公式" → 只生成这1个节点的讲义 → 1500字 → 刚好读完 ✅
  用户点击"压杆稳定" → 再生成这1个节点的讲义 → 2500字 → 按需展开 ✅
```

### 三个核心原则

1. **按需生成**：用户问哪个节点，就只生成哪个节点的讲义
2. **上下文感知**：讲义内容根据节点在树中的位置自动调整深度
3. **边界清晰**：讲义范围 = 本节点 + 直接子节点，不越界展开

### 两大特色 ⭐

1. **知识图谱融合**：在讲义中嵌入当前节点与其它节点的关联（因果、对比、依赖），让知识不再是孤岛
2. **小白友好叙事**：每个知识点都回答"为什么学这个"、"从哪来的"、"能干啥"，用生活类比代替抽象定义

## Step 1: 节点定位

### 定位方式

| 方式 | 示例 | 处理逻辑 |
|-----|------|---------|
| 精确名称 | "讲一下欧拉公式" | 直接搜索匹配 |
| 层级路径 | "第十章压杆稳定的欧拉公式" | 按路径逐级定位 |
| 模糊搜索 | "那个临界压力的公式" | 语义匹配最近节点 |
| 序号定位 | "第5个节点" | 按导图顺序编号 |

### 定位算法

```python
def locate_node(knowledge_tree, query: str) -> dict:
    """在知识树中定位用户请求的节点"""

    # 1. 精确匹配
    for node in flatten_tree(knowledge_tree):
        if node['text'] == query:
            return node

    # 2. 包含匹配
    for node in flatten_tree(knowledge_tree):
        if query in node['text'] or node['text'] in query:
            return node

    # 3. 路径匹配（如"第十章/欧拉公式"）
    if '/' in query:
        parts = query.split('/')
        current = knowledge_tree
        for part in parts:
            for child in current.get('children', []):
                if part in child['text']:
                    current = child
                    break
        return current

    # 4. 模糊匹配（找最相似的）
    return find_most_similar(knowledge_tree, query)
```

## Step 2: 确定讲义参数

### 层级-讲义映射规则

节点在知识树中的层级**决定了讲义的深度和范围**：

| 层级 | 代表什么 | 讲义风格 | 默认字数 | 讲义重点 |
|-----|---------|---------|---------|---------|
| **L1** 根节点 | 整本书 | 全书概览 | 800-1200字 | 全书框架、章节关系、学习路线 |
| **L2** 部分/篇 | 主题模块 | 模块导览 | 600-1000字 | 本部分核心问题、章节递进逻辑 |
| **L3** 章 | 一章内容 | 章节讲义 | 1500-2500字 | 概念引入→核心内容→例题→总结 |
| **L4** 节/小节 | 一个知识点 | 知识点讲义 | 800-1500字 | 定义→推导→公式→应用→注意 |
| **L5** 细节 | 具体要点 | 要点卡片 | 300-600字 | 核心结论→记忆口诀→常见错误 |

### 层级示例（材料力学）

```
L1: 材料力学 → 全书概览（800字）
L2: 基本变形 → 模块导览（800字，介绍拉压/扭转/弯曲的关系）
L3: 第二章 拉伸压缩 → 章节讲义（2000字，完整教学流程）
L4: 胡克定律 → 知识点讲义（1000字，定义→公式→条件→应用）
L5: 比例极限 → 要点卡片（400字，结论+口诀+易错点）
```

### 知识性质-讲义风格调整

| 知识性质 | 讲义调整 |
|---------|---------|
| 🔴 重点 | 增加"为什么是重点"、"考试怎么考" |
| 🟠 难点 | 增加"为什么难"、"理解突破口" |
| 🟡 易错点 | 增加"典型错误示例"、"避坑口诀" |
| 🟢 基础 | 增加"后续哪里用到"、"学完能做什么" |
| 🟣 考点 | 增加"历年考频"、"真题链接" |

## Step 3: 讲义撰写六大特色 ⭐

### 特色一：知识图谱融合

讲义不只是孤立地讲一个知识点，而是**把它放在知识网络中**——告诉读者这个节点和谁有关系、什么关系。

```markdown
## 🔗 知识关联

本知识点与以下内容有联系：

| 关联方向 | 关联内容 | 关系类型 | 一句话说明 |
|---------|---------|---------|-----------|
| ← 前置 | 截面法 | 依赖 | 求内力是求应力的前提 |
| ← 前置 | 轴力计算 | 依赖 | 先算出FN才能算σ |
| → 后续 | 胡克定律 | 因果 | 有了应力才能算变形 |
| ↔ 对比 | 切应力τ | 对比 | σ垂直截面，τ平行截面 |
| → 后续 | 强度校核 | 应用 | σ≤[σ]是设计依据 |
```

**实现逻辑**：从 book-mindmap 生成的知识关联图中，提取当前节点的所有关联（入向+出向），按"前置→当前→后续"排列，让读者看到知识从哪来、到哪去。

### 特色二：小白友好叙事（为什么-从哪来-能干啥）

每个知识点都回答三个问题，让完全不懂的人也能建立直觉：

| 问题 | 对应讲义模块 | 示例（欧拉公式） |
|-----|------------|----------------|
| **为什么要学这个？** | 动机与背景 | 桥梁柱子明明没断却塌了——这就是失稳 |
| **这个概念从哪来的？** | 历史与起源 | 欧拉1757年推导，比法国大革命还早32年 |
| **学了能干啥？** | 用途与价值 | 设计细长柱子时确保不会"突然弯掉" |

**撰写原则**：
- **不用术语解释术语**：❌ "临界压力是使压杆处于临界状态的压力" → ✅ "压力大到某个值，杆子突然弯了，这个值就是临界压力"
- **先给直觉再给定义**：先让读者"感觉到"这个概念在说什么，再给出精确定义
- **用生活类比打开话题**：压杆失稳 ≈ 站在易拉罐上踩扁它；应力集中 ≈ 撕包装袋时先剪个小口

### 特色三：历史背景与人物故事

为关键知识点补充历史背景，让知识有温度：

```markdown
## 📜 来龙去脉

**1757年**，欧拉（Leonhard Euler）在研究柱的弯曲问题时，
推导出了临界压力公式。有趣的是，欧拉本人更关注数学优美性，
并未意识到这个公式的工程价值。

直到**19世纪工业革命**，大量铁路桥梁坍塌事故（如1879年
Tay Bridge灾难）让人们意识到：杆件不是"不断就行"，
还要"不弯"。欧拉公式才从纯数学走进工程设计。

💡 这告诉我们：**强度和稳定是两回事**——材料没断，
但形状变了，一样会出事。
```

**历史信息来源**：优先从书中提取，补充知识使用网络搜索。若无法获取准确历史信息，标注"历史背景待补充"，不编造。

### 特色四：概念类比库

为抽象概念提供生活类比，帮助建立直觉：

| 学科 | 抽象概念 | 生活类比 |
|-----|---------|---------|
| 材料力学 | 应力集中 | 撕包装袋先剪小口——应力集中在缺口处 |
| 材料力学 | 压杆失稳 | 易拉罐站上去踩扁——不是断了而是弯了 |
| 材料力学 | 弯矩正负 | 微笑😊=正弯矩（下凸），皱眉😦=负弯矩（上凸） |
| 数据结构 | 栈 | 叠盘子——最后放的最先拿 |
| 数据结构 | 队列 | 排队买票——先来先服务 |
| 编程 | 闭包 | 背包——走了还能拿到出发地装的东西 |
| 编程 | 递归 | 俄罗斯套娃——打开一个还有一个小号的 |

**撰写规则**：
- 每个L4及以上知识点至少提供1个类比
- 类比必须来自日常生活，不能是另一个专业术语
- 类比只帮助建立直觉，不能替代精确定义——类比后必须跟一句"但严格来说..."

### 特色五：前后文衔接

讲义开头和结尾都要"接上"上下文，不让读者感到突兀：

```markdown
# 开头衔接
> 在上一节"轴力计算"中，我们学会了求杆件内部力的大小。
> 但光知道力有多大还不够——力集中在多大面积上？
> 这就是**应力**要回答的问题。

# 结尾衔接
> 现在我们知道了怎么算应力。但只知道"有多大"还不够——
> 变形有多大？能不能恢复？下一节"胡克定律"将回答这个问题。
```

### 特色六：关联节点推荐

讲义末尾不只是列出子节点，而是根据关联类型给出**有理由的推荐**：

```markdown
💡 **推荐继续探索**：

| 方向 | 节点 | 为什么推荐 |
|-----|------|-----------|
| 📌 自然深入 | 长度系数μ | 欧拉公式中μ怎么取？这是核心参数 |
| 🔗 关联延伸 | 柔度判别 | 欧拉公式不是万能的——哪些杆能用？ |
| 🔗 对比理解 | 强度校核 | 稳定 vs 强度，两套不同的校核逻辑 |
| ⬆️ 返回上级 | 压杆稳定 | 回到本章总览 |
```

## Step 4: 边界控制（关键机制）

### 边界规则

| 规则 | 说明 | 示例 |
|-----|------|------|
| **本节点必展开** | 讲义必须包含本节点的完整内容 | "胡克定律"讲义必须讲清σ=Eε |
| **子节点只点名** | 列出子节点名称，但不深入展开 | 提到"比例极限"但不解释细节 |
| **不越界到兄弟** | 不主动展开同级的其他节点 | 讲"胡克定律"不讲"应力集中" |
| **父节点给路径** | 只提供父节点路径定位，不重复内容 | 标注"位于：拉伸压缩→胡克定律" |

### 边界控制示例

```
用户请求："展开 胡克定律"

✅ 应该包含：
  - 胡克定律的定义（本节点）
  - σ=Eε 的公式和含义（本节点）
  - 适用条件的说明（本节点）
  - 提到"比例极限"（子节点点名）
  - 提到"弹性极限"（子节点点名）

❌ 不应包含：
  - 比例极限的详细解释（那是L5的内容，需要单独展开）
  - 应力集中的讲解（兄弟节点，需要单独请求）
  - 轴力图的画法（同章其他节，需要单独请求）
```

  → "展开 弹性极限" —— 无残余变形的极限

⬆️ **返回上级**：

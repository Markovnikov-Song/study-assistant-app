## 一、项目概述

### 1.1 产品定位

**Learning OS** 是一款 AI 驱动的多端学习助手应用，定位为"学习专用操作系统"。核心理念是 **"纯手动永远可用、AI 叠加而非替代"**，提供四种递进使用模式：

| 模式 | 说明 |
|------|------|
| 纯手动 | 所有功能手动操作，无需 AI |
| DIY | 用户自主上传资料、创建计划 |
| Skill 驱动 | 通过学习方法论 Skill 自动化学习流程 |
| Multi-Agent | 多 Agent 协作自动管理学习全流程 |

### 1.2 技术架构总览

```
┌─────────────────────────────────────────────────┐
│              Flutter 前端 (多端)                  │
│  Riverpod · GoRouter · Dio · EventBus            │
├─────────────────────────────────────────────────┤
│              FastAPI 后端 (Python)                │
│  SQLAlchemy · PGVector · LLM Service · MCP Layer  │
├─────────────────────────────────────────────────┤
│              PostgreSQL + PGVector                │
└─────────────────────────────────────────────────┘
```

- **前端**：Flutter 3.11+，支持 Android / iOS / macOS / Windows / Linux / Web
- **后端**：Python 3.10+ / FastAPI
- **数据库**：PostgreSQL + PGVector（向量检索）
- **AI**：OpenAI 兼容 API（支持多模型配置：轻量/重型/视觉/嵌入模型）

### 1.3 四大核心模块（底部导航 Tab）

| 模块 | 路由 | 核心功能 |
|------|------|---------|
| 💬 答疑室 | `/` | RAG 问答（strict/broad/hybrid/solve）、流式输出、会话管理 |
| 🏫 图书馆 | `/course-space` | 学科列表 → 课程空间（大纲） → 思维导图 → 节点讲义 → 练习 → 错题收录 |
| 🛠 工具箱 | `/toolkit` | 复盘中心、笔记本、解题、出题、脑图工坊、方法库、学习日历（7 个子工具） |
| 👤 我的 | `/profile` | 学生证（编辑资料）、学科管理、资料管理、对话历史、背景风格、Learning OS 模式切换 |

## 二、用户需求分析

### 2.1 目标用户

Learning OS 的目标用户是**中国在校学生**（初中至大学），核心特征：

| 用户特征 | 描述 |
|----------|------|
| 学习资料分散 | 教材、PPT、PDF、历年题散落各处，缺乏统一管理 |
| 有学习方法意识但不会用 | 知道费曼法、间隔重复等概念，但缺乏执行能力 |
| 多学科并行 | 同时备考多门课程，缺乏跨学科的时间规划能力 |
| 依赖 AI 但担心过度 | 愿意用 AI 辅助学习，但不希望 AI 替代自己的思考过程 |
| 移动端为主 | 主要在手机/平板上学习，需要随时可用 |

### 2.2 核心痛点

| 编号 | 痛点 | 严重度 | 现有解决方案的问题 |
|------|------|--------|-------------------|
| P1 | **知识遗忘无追踪** | 高 | 错题记了不复习，复习没有科学间隔，靠感觉安排 |
| P2 | **资料管理混乱** | 高 | 文件夹堆满了 PDF，找不到想看的内容，不同平台割裂 |
| P3 | **学习方法论难落地** | 中 | 知道费曼法好，但每次都嫌麻烦直接跳过 |
| P4 | **刷题缺乏针对性** | 中 | 刷大量题但不知道自己真正薄弱的知识点在哪里 |
| P5 | **学习缺乏系统性规划** | 中 | 没有人帮忙安排学习节奏，全凭自律 |
| P6 | **AI 回答不可靠** | 中 | 通用 AI 会编造知识，希望基于自己的资料回答 |
| P7 | **跨学科管理难** | 低 | 各科学习进度、计划、资料各自为政 |

### 2.3 用户故事

#### 答疑室（知识获取）

```
作为一个正在预习物理的学生，
我希望基于自己的教材讲义向 AI 提问（strict 模式），
以便得到仅来自我的学习资料的准确回答，而不是 AI 编造的内容。

验收标准：
- AI 回答必须标注知识来源（出自哪个文件的哪段内容）
- 找不到相关内容时，明确告知"资料库中未找到"
- 支持拍照上传题目进行 OCR 识别后提问
```

```
作为一个正在复习的学生，
我希望 AI 能结构化地解题（solve 模式），
输出考点、解题思路、步骤、踩分点和易错点，
以便我不仅知道答案，还知道怎么想到这个答案。
```

#### 图书馆（知识体系化）

```
作为一个需要系统掌握某门课程的学生，
我希望上传教材/PPT 后自动生成思维导图，
每个节点能展开为详细讲义，
以便我对整门课的知识结构一目了然。

验收标准：
- 思维导图基于我的资料生成（RAG 驱动）
- 节点可以标注知识性质（重点/难点/考点/基础）
- 讲义支持流式生成，不阻塞阅读
- 节点点亮状态记录阅读进度
```

```
作为一个考前冲刺的学生，
我希望基于知识节点生成练习题，
控制出题范围在前置/当前/后置三个区域，
以便检验我对当前知识点的掌握程度。

验收标准：
- 题目难度可选（基础/中等/进阶/混合）
- 错题自动收录到错题本
- 答题后可以查看解析
```

#### 工具箱（学习闭环）

```
作为一个需要反复巩固薄弱知识点的学生，
我希望错题能自动进入 SM-2 复习队列，
在即将遗忘的时间点提醒我复习，
以便用最少的复习次数达到长期记忆效果。

验收标准：
- 错题自动创建复习卡片
- 复习间隔根据我的记忆表现动态调整
- 学科差异化（数学比英语忘得快）
- 每日复习队列和过期任务提醒
```

```
作为一个有很多学习方法论经验贴的学生，
我希望把"费曼法"、"间隔重复"这类文字描述转化为可执行的 AI 工作流（Skill），
以便一键启动，让 AI 按方法论的步骤引导我学习。

验收标准：
- 可以对话式创建 Skill
- Skill 可以从市场下载
- Skill 执行时按步骤引导，每步调用对应工具
```

```
作为一个备考的学生，
我希望用日历规划每天的学习任务，
追踪学习时长和打卡天数，
查看各学科的时间投入占比，
以便合理安排复习节奏。

验收标准：
- 支持创建单次事件和重复例程
- 学习 Session 计时记录
- 统计面板展示 7 天/30 天趋势
```

#### 4.2.3 RAG 流水线
```

    → 相关度判断（阈值可配）
    → strict: 仅资料库 / broad: 资料库+通用 / hybrid: 优先资料库
    → LLM 生成回答（系统提示词根据模式切换）
    → 返回回答 + 知识来源
```

#### 4.2.4 LLM 服务
- OpenAI 兼容 API（可配置任意提供商）
- 支持三种模型：轻量（`fast`）、重型（`heavy`）、视觉（`vision`）
- 场景化 max_tokens/temperature 配置
- 流式输出支持（SSE）

#### 4.2.5 SM-2 改良算法
- **评分简化**：0-3 分（忘了/模糊/想起/巩固）
- **学科差异化衰减**：数学 0.85、英语 1.15、历史 0.95 等
- **题目难度系数**：难题需要更频繁复习
- **遗忘惩罚**：连续错误降低期望值
- **掌握度量化**：0-5 分追踪

## 六、设计原则总结

| 原则 | 体现 |
|------|------|
| 骨架优先 | CAS 框架设计文档先行，再编码实现 |
| 单向依赖 | 前端 → 后端 → 数据库，无循环依赖 |
| 可插拔 | LLM 可切换、MCP 工具可插拔、Skill 可扩展 |
| 渐进式重构 | `api/` 兼容层保留，新功能在 `backend/` 开发 |
| AI 不可用时降级 | LLM 失败走硬编码兜底，系统永远可用 |
| 提示词配置化 | YAML 管理，硬编码 fallback，`PromptManager` 统一加载 |

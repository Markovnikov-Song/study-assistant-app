# 🎯 如何运行 release.ps1 脚本

## 方法一：右键运行（最简单）

### 步骤 1：找到脚本文件
在文件资源管理器中，打开项目文件夹：
```
C:\Users\41100\develop\study_assistant_app
```

找到 `release.ps1` 文件

### 步骤 2：右键运行
1. **右键点击** `release.ps1` 文件
2. 选择 **"使用 PowerShell 运行"** 或 **"Run with PowerShell"**

⚠️ **注意**：这样运行会缺少参数，脚本会报错。所以推荐用方法二。

---

## 方法二：在终端中运行（推荐）

### 步骤 1：打开 PowerShell

**方式 A - 在文件夹中打开：**
1. 在文件资源管理器中打开项目文件夹
   ```
   C:\Users\41100\develop\study_assistant_app
   ```
2. 在地址栏输入 `powershell` 然后按回车
   
   或者：
   
3. 在空白处 **按住 Shift + 右键**
4. 选择 **"在此处打开 PowerShell 窗口"** 或 **"在终端中打开"**

**方式 B - 手动进入目录：**
1. 按 `Win + X`，选择 **"Windows PowerShell"** 或 **"终端"**
2. 输入命令进入项目目录：
   ```powershell
   cd C:\Users\41100\develop\study_assistant_app
   ```

### 步骤 2：运行脚本

在 PowerShell 窗口中输入：

```powershell
.\release.ps1 -Version "1.2.0" -BuildNumber 3 -Changelog "✨ 新增 API 配置功能"
```

然后按回车！

### 如果提示"无法加载脚本"

先运行这个命令（允许运行脚本）：
```powershell
Set-ExecutionPolicy -Scope Process -ExecutionPolicy Bypass
```

然后再运行发布脚本。

## 方法三：使用 VS Code（如果你用 VS Code）

### 步骤 1：打开项目

## 🎬 运行后会发生什么？

脚本会自动：
1. ✅ 更新版本号
2. ✅ 清理旧文件
3. ✅ 下载依赖
4. ✅ 构建 APK（这一步需要几分钟）
5. ✅ 复制 APK 文件
6. ✅ 更新配置
7. ✅ 提交到 Git
8. ❓ 询问：**是否推送到 GitHub? (y/n):**
   - 输入 `y` 然后回车 = 推送
   - 输入 `n` 然后回车 = 不推送

## ⚠️ 常见问题

### Q: 提示"无法识别 flutter 命令"
**A:** Flutter 没有安装或没有添加到 PATH
```powershell
# 检查 Flutter 是否安装
flutter --version
```

### Q: 提示"无法加载脚本"
**A:** 需要允许运行脚本
```powershell
Set-ExecutionPolicy -Scope Process -ExecutionPolicy Bypass
```

### Q: 提示"找不到路径"
**A:** 确保在正确的目录
```powershell
# 查看当前目录
pwd

# 应该显示：C:\Users\41100\develop\study_assistant_app
```

## 🆘 如果还是不会

**最简单的方法 - 手动发布：**

1. **打开 PowerShell**（任意方式）

2. **复制粘贴以下所有命令**（一次性全部复制）：

```powershell
cd C:\Users\41100\develop\study_assistant_app
flutter clean
flutter pub get
flutter build apk --release
New-Item -ItemType Directory -Force -Path backend\downloads
Copy-Item build\app\outputs\flutter-apk\app-release.apk backend\downloads\app-v1.2.0.apk
```

3. **等待构建完成**（几分钟）

4. **完成！** APK 文件在：`backend\downloads\app-v1.2.0.apk`

然后手动：
- 更新 `pubspec.yaml` 的版本号
- 更新 `backend\.env` 的版本配置
- 提交到 Git

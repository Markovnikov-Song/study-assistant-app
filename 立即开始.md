# 立即开始 - 后台保活方案

## 第一步：安装依赖

```powershell
flutter pub get
```

如果遇到错误，运行：
```powershell
flutter clean
flutter pub get
```

## 第二步：修复服务器更新日志乱码（可选）

```powershell
.\quick_fix_encoding.ps1
```

输入服务器密码，等待完成。

## 第三步：构建新版本

```powershell
flutter clean
flutter build apk --release
```

构建完成后，APK 位于：
```
build/app/outputs/flutter-apk/app-release.apk
```

## 第四步：测试

### 测试 AI 输出保活

1. 安装新 APK 到手机
2. 打开应用，进入聊天
3. 发送问题，AI 开始输出
4. **立即切换到微信**
5. 等待 20 秒
6. 切回应用
7. ✅ 应该看到完整回答

### 测试应用更新保活

1. 触发更新检查
2. 点击"立即更新"
3. **立即切换到其他应用**
4. 下拉通知栏，查看下载进度
5. 等待下载完成
6. ✅ 自动弹出安装界面

## 第五步：发布新版本（可选）

如果测试通过，可以发布新版本：

```powershell
# 更新版本号为 1.2.1
.\release.ps1 -Version "1.2.1" -BuildNumber 4 -Changelog "🔧 优化后台保活，AI 输出和下载不再中断"

# 部署到服务器
.\deploy_to_server.ps1 -Version "1.2.1"
```

然后创建 GitHub Release：
1. 访问：https://github.com/Markovnikov-Song/study-assistant-app/releases/new
2. Tag: `v1.2.1`
3. 标题：`v1.2.1 - 后台保活优化`
4. 上传 APK：`backend/downloads/app-v1.2.1.apk`
5. 发布

## 常见问题

### Q: flutter pub get 失败？
```powershell
flutter clean
flutter pub get
```

### Q: 构建失败？
检查错误信息，可能需要：
```powershell
flutter doctor
```

### Q: 还是被杀后台？
需要用户在手机设置中：
- 小米：设置 → 应用管理 → 伴学 → 省电策略 → 无限制
- 华为：设置 → 应用启动管理 → 伴学 → 手动管理
- OPPO：设置 → 电池 → 应用耗电管理 → 伴学 → 允许后台运行

详细说明见：`防止后台被杀说明.md`

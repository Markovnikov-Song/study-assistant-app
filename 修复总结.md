## 问题 2：切换应用后被杀后台 ✅ 已配置

### 问题描述
用户切换到其他应用后，返回时应用被系统杀掉，需要重新启动。

### 解决方案

#### 已完成的配置
修改了 `android/app/src/main/AndroidManifest.xml`：

1. **新增权限**：
   ```xml
   <uses-permission android:name="android.permission.WAKE_LOCK"/>
   <uses-permission android:name="android.permission.FOREGROUND_SERVICE"/>
   ```

2. **Application 配置**：
   ```xml
   android:allowBackup="true"
   android:supportsRtl="true"
   android:persistent="false"
   ```

3. **Activity 配置**：
   ```xml
   android:excludeFromRecents="false"      <!-- 保留在最近任务 -->
   android:alwaysRetainTaskState="true"    <!-- 保持任务状态 -->
   android:clearTaskOnLaunch="false"       <!-- 不清除任务 -->
   android:stateNotNeeded="false"          <!-- 需要保存状态 -->
   ```

#### 重新构建应用
配置修改后需要重新构建：

```powershell
# 清理
flutter clean

# 获取依赖
flutter pub get

# 构建 APK
flutter build apk --release
```

新的 APK 将位于：`build/app/outputs/flutter-apk/app-release.apk`

#### 用户端配置建议
由于不同手机厂商的后台管理策略不同，建议用户：

**小米手机**：
- 设置 → 应用设置 → 应用管理 → 伴学 → 省电策略 → 无限制
- 设置 → 应用设置 → 应用管理 → 伴学 → 自启动 → 开启

**华为手机**：
- 设置 → 应用 → 应用启动管理 → 伴学 → 手动管理
- 允许自启动、允许关联启动、允许后台活动

**OPPO 手机**：
- 设置 → 电池 → 应用耗电管理 → 伴学 → 允许后台运行
- 设置 → 应用管理 → 伴学 → 权限 → 自启动 → 开启

**VIVO 手机**：
- 设置 → 电池 → 后台高耗电 → 伴学 → 允许
- 设置 → 更多设置 → 权限管理 → 自启动 → 伴学 → 开启

详细说明请查看：`防止后台被杀说明.md`
